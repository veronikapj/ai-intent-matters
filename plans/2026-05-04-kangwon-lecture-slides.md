**S09 | 왜 딸기의 r을 못 셌을까? (Tokenization)**
```
타입: 코드 / 시각화 대비
---
[우리가 보는 것: 문자열(String)]
['s', 't', 'r', 'a', 'w', 'b', 'e', 'r', 'r', 'y'] → "r이 3개네!"

[AI가 보는 것: 토큰(Token)]
[Token ID: 49323] → "이게 무슨 글자로 이루어져 있는지는 모름"

딸기 문제를 못 푼 건 AI가 '멍청해서'나 '의도를 몰라서'가 아닙니다.
LLM은 텍스트를 문자가 아닌 '토큰(정수 ID)' 단위로 쪼개서 보기 때문에 발생하는 "단순한 기술적 한계"입니다.
```

---
*(TMI: 이 기술적 한계를 극복하기 위해 만들어진 OpenAI o1 모델의 개발 프로젝트명이 진짜 'Strawberry'였습니다)*

단순히 지능을 높인 게 아니라, AI에게 '도구'와 '시간'을 주었습니다.
1. 코드 실행: 속으로 파이썬(Python) 코드를 짜서 실제로 `len([c for c in "strawberry" if c == 'r'])`을 돌려보게 함
2. 사고 과정(CoT): 답을 바로 뱉지 않고, 숨겨진 생각 과정 속에서 스스로 단어를 쪼개보고 틀리면 다시 검증하게 함

**S11 | 진짜 본질적인 한계 (Deep Intent)**
```
타입: 텍스트 대비
---
진짜 본질적이고 어려운 한계는 기술로 해결되지 않는 "깊은 의도(Deep Intent)"입니다.

- 표면 의도: "테스트 통과시켜줘" (완벽하게 이해함)
- 깊은 의도(암묵적 제약): "코드를 고쳐서, 다른 기능은 건드리지 말고" (말하지 않았으므로 모름)

이것이 앞서 본 굿하트의 법칙(Goodhart's Law)이 무서운 이유입니다.
AI는 "당신이 원하는 것"이 아니라 "당신이 말한 것"만 무지성으로 최적화합니다.

스피커 노트:
"앞으로 AI가 발전하면 내 마음을 다 알아줄까요? 분명 나아지겠지만 근본적인 갭은 남습니다. 왜냐하면 의도는 종종 말하지 않은 것, 아직 정의하지 않은 것, 심지어 여러분 자신도 완벽히 모르는 것에 있기 때문입니다. 의도는 결국 여러분의 머릿속에 있고, AI는 그것을 흉내낼 수는 있어도 소유할 수는 없습니다."
```
